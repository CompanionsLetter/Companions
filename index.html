Ce sunt subprogramele?

DEFINIȚIE SIMPLĂ

Un subprogram (sau funcție) este un bloc de cod căruia îi dai un nume și pe care îl poți apela (folosi) de oricâte ori în program, fără să rescrii același cod.

Gândești-te la o funcție ca la o rețetă: o scrii o dată, o numești, și o "gătești" (apelezi) de câte ori ai nevoie.

De ce folosim subprograme?

  • 🔁 Reutilizare — scriem codul o dată, îl folosim de mai multe ori
  • 🧩 Organizare — programul e mai ușor de citit și înțeles
  • 🐛 Depanare — găsim erorile mai ușor
  • 📦 Modularizare — descompunem problema în bucăți mici

Structura unui program C++ cu funcții

structura_generala.cpp 
#include <iostream>
using namespace std;

// 1. Definim functia INAINTE de main()
int suma(int a, int b) {
    return a + b;
}

// 2. Programul principal
int main() {
    int rezultat = suma(3, 5); // 3. Apelam functia
    cout << rezultat << endl;   // Afiseaza: 8
    return 0;
}
Rezumat rapid: Funcția e definită înainte de main(), iar în main() o apelăm folosind numele ei.

Subprograme cu tip și fără tip

1. Cele două tipuri de funcții

Caracteristică Cu tip (returnează valoare) Fără tip — void
Sintaxă antet int/float/... numeF(...) void numeF(...)

2. Funcție CU tip — sintaxă completă

tip_returnat nume_functie(tip param1, tip param2, ...) {
    // variabile locale si calcule
    return rezultat;
}

Fiecare parte a acestei sintaxe are un rol:

  • tip_returnat — ce tip de valoare întoarce funcția: int, float, double, char, bool
  • nume_functie — numele cu care o apelăm (ex: suma, maxim, cmmdc)
  • parametri — datele de intrare (pot lipsi)
  • return — trimite rezultatul înapoi la apelant

4. Parametri formali vs Parametri actuali

DIFERENȚA ESENȚIALĂ

Parametri formali = variabilele din antetul funcției (placeholdere)

Parametri actuali = valorile reale pe care le trimitem la apel

// a si b sunt PARAMETRI FORMALI (placeholdere)
int suma(int a, int b) {
    return a + b;
}

int main() {
    int x = 5, y = 3;
    // x si y sunt PARAMETRI ACTUALI (valorile reale)
    cout << suma(x, y);    // a=5, b=3 → returnează 8
    cout << suma(10, 20); // a=10, b=20 → returnează 30
}

5. Transmiterea prin valoare vs prin referință

Prin VALOARE (normal)

Funcția primește o copie a valorii. Modificările din funcție nu afectează variabila originală.

void f(int x) {
    x = 100; // schimba copia
}
int a = 5;
f(a);
// a ramane 5!

Prin REFERINȚĂ (&)

Funcția primește adresa variabilei. Modificările afectează variabila originală.

void f(int &x) {
    x = 100; // schimba original
}
int a = 5;
f(a);
// a devine 100!

Exemplu clasic: Schimbarea a două variabile (swap)

// GRESIT - prin valoare (nu functioneaza)
void schimba_gresit(int a, int b) {
    int temp = a; a = b; b = temp; // se modifica doar copiile!
}

// CORECT - prin referinta
void schimba(int &a, int &b) {
    int temp = a; // salvam valoarea lui a
    a = b;        // a primeste valoarea lui b
    b = temp;     // b primeste valoarea veche a lui a
}

int main() {
    int x = 7, y = 1;
    cout << x << " " << y << endl; // 7 1
    schimba(x, y);
    cout << x << " " << y << endl; // 1 7
}

6. Declarații anticipate (prototipuri)

Dacă vrei să definești funcția DUPĂ main(), trebuie să declari prototipul înainte:

#include <iostream>
using namespace std;

int suma(int, int); // PROTOTIP - anuntam compilatorul

int main() {
    cout << suma(3, 4); // putem folosi functia
    return 0;
}

// Definim functia dupa main
int suma(int a, int b) {
    return a + b;
}
GREȘELI FRECVENTE
  • ❌ Uiți return la o funcție cu tip
  • ❌ Folosești return valoare; într-o funcție void
  • ❌ Numărul de parametri la apel ≠ numărul de parametri formali
  • ❌ Tipurile parametrilor nu corespund
  • ❌ Folosești variabile locale în afara funcției unde au fost declarate
Tema 6

Domenii de vizibilitate & Efecte colaterale

1. Cele trei tipuri de variabile

Tip Unde se declară Unde e vizibilă Durată în memorie
Globală În afara oricărei funcții Tot programul Toată execuția programului
Locală Înăuntrul unei funcții Doar în funcția respectivă Doar cât durează execuția funcției
Parametru formal În antetul funcției Doar în corpul funcției Doar cât durează execuția funcției

Exemplu vizual

#include <iostream>
using namespace std;

int global = 10; // ← VARIABILA GLOBALA: vizibila peste tot

void functie(int param) { // param = parametru formal
    int locala = 5;   // ← VARIABILA LOCALA: vizibila doar in functie()
    cout << global;  // OK - variabila globala e vizibila
    cout << locala;  // OK - suntem in functie()
    cout << param;   // OK - parametru vizibil in corpul functiei
}

int main() {
    cout << global;  // OK
    // cout << locala;  // EROARE! locala nu exista in main()
    functie(3);
    return 0;
}

2. Variabile globale — avantaje și pericole

AVANTAJ: Comunicare simplă

Variabilele globale pot fi accesate și modificate din orice funcție, fără a le pasa ca parametri.

PERICOL: Efecte colaterale

Un efect colateral = o funcție modifică o variabilă globală sau un parametru prin referință în mod neașteptat.

Exemplu de efect colateral — cum funcțiile se "influențează"

int c = 2; // variabila globala

int f(int a) {
    c++;        // EFECT COLATERAL: modifica variabila globala c!
    return c * a;
}

int main() {
    cout << f(c) << endl; // c devine 3, returneza 3*2=6
    cout << f(c) << endl; // c devine 4, returneza 4*3=12
    // Fiecare apel schimba c - comportament "surprinzator"!
}

3. Variabile locale — NU se inițializează automat

Atenție! Variabilele locale nu au valoare inițială automată. Conțin "gunoi" din memorie dacă nu le inițializăm explicit. 
void exemplu() {
    int x;          // x contine "gunoi" (valoare aleatoare)
    int y = 0;      // corect: y = 0
    cout << x;      // PERICOL: afiseaza valoare necunoscuta
    cout << y;      // OK: afiseaza 0
}

Exemplu: matrice cu funcții

#include <iostream>
using namespace std;
const int MAX = 100;

// Citire matrice
void citire(int t[MAX][MAX], int &n, int &m) {
    cout << "Linii: "; cin >> n;
    cout << "Coloane: "; cin >> m;
    for (int i=0; i<n; i++)      // parcurgem liniile
        for (int j=0; j<m; j++)  // parcurgem coloanele
            cin >> t[i][j];          // citim elementul
}

// Afisare matrice
void afisare(int t[MAX][MAX], int n, int m) {
    for (int i=0; i<n; i++) {
        for (int j=0; j<m; j++)
            cout << t[i][j] << " ";
        cout << endl; // rand nou dupa fiecare linie
    }
}

// Maximul din matrice
int maxim(int t[MAX][MAX], int n, int m) {
    int max_val = t[0][0]; // incepem cu primul element
    for (int i=0; i<n; i++)
        for (int j=0; j<m; j++)
            if (t[i][j] > max_val) max_val = t[i][j];
    return max_val;
}

int main() {
    int mat[MAX][MAX], n, m;
    citire(mat, n, m);
    cout << "Matricea:\n";
    afisare(mat, n, m);
    cout << "Max = " << maxim(mat, n, m);
    return 0;
}

#include <iostream>
#include <cstring>   // pentru strlen()
using namespace std;

void inversare(char s[]) {
    int n = strlen(s); // n = lungimea sirului
    for (int i=0; i<n/2; i++) {  // mergem pana la mijloc
        char temp = s[i];
        s[i] = s[n-1-i];          // primul ↔ ultimul
        s[n-1-i] = temp;
    }
}

int main() {
    char cuvant[100];
    cout << "Scrie un cuvant: ";
    cin >> cuvant;
    inversare(cuvant);
    cout << "Inversat: " << cuvant << endl;
    return 0;
}

Tema 8

Subprograme recursive

1. Ce este recursivitatea?

DEFINIȚIE SIMPLĂ

O funcție este recursivă dacă se apelează pe sine însuși. E ca o oglindă cu oglinzi — rezultatul unui pas depinde de un pas mai mic.

Orice funcție recursivă are OBLIGATORIU două părți:

1. Cazul de bază (stop)

Condiția care oprește recursivitatea. Fără el, funcția ar rula la infinit → program crash.

2. Apelul recursiv

Funcția se apelează pe sine cu un argument mai mic/simplu, apropiindu-se de cazul de bază.

tip nume_f(parametri) {
    if (caz_de_baza) // STOP — returnăm direct
        return valoare_simpla;
    else // RECURSIV — ne apelăm cu caz mai mic
        return nume_f(parametri_mai_mici);
}

2. Exemplu clasic: Factorial

Matematică: n! = n × (n-1) × (n-2) × ... × 1, iar 0! = 1

factorial.cpp
1 long int factorial(int n) { long int — pentru numere mari
2 if (n == 0) CAZ DE BAZĂ: 0! = 1
3 return 1; Oprire: returnăm 1
4 else CAZ RECURSIV
5 return n * factorial(n - 1); n × factorial cu n-1 (mai mic!)
6 }
CUM FUNCȚIONEAZĂ — PAS CU PAS

factorial(4) → 4 × factorial(3)
  → 4 × 3 × factorial(2)
    → 4 × 3 × 2 × factorial(1)
      → 4 × 3 × 2 × 1 × factorial(0)
        → 4 × 3 × 2 × 1 × 1 = 24 ✅

3. Exemplu: Suma 1+2+3+...+n

int suma(int n) {
    if (n == 0)            // caz de baza: suma pana la 0 este 0
        return 0;
    else
        return n + suma(n-1); // n + suma pana la n-1
}
// suma(4) = 4 + suma(3) = 4 + 3 + suma(2) = ... = 4+3+2+1+0 = 10

4. Exemplu: Numărul de cifre

int cifre(int n) {
    if (n == 0)            // caz de baza: 0 are 0 cifre
        return 0;
    else
        return 1 + cifre(n/10); // 1 cifra + cifrele din n/10
}
// cifre(1234) = 1 + cifre(123) = 2 + cifre(12) = 3 + cifre(1) = 4 + cifre(0) = 4

5. Exemplu: Șirul Fibonacci

Șirul: 1 1 2 3 5 8 13 21 ... Fiecare termen = suma celor două precedente

int fib(int n) {
    if (n == 0) return 1; // cazuri de baza
    if (n == 1) return 1;
    else return fib(n-1) + fib(n-2); // suma celor doua precedente
}
// fib(4) = fib(3) + fib(2) = (fib(2)+fib(1)) + (fib(1)+fib(0)) = (2+1)+(1+1) = 5

6. CMMDC recursiv

// Varianta 1: prin scaderi repetate
int cmmdc(int a, int b) {
    if (a*b == 0) return a+b; // caz de baza: unul e 0
    if (a > b) return cmmdc(a-b, b);
    else return cmmdc(a, b-a);
}

// Varianta 2: prin impartiri (mai eficienta)
int cmmdc2(int a, int b) {
    if (b == 0) return a; // caz de baza
    return cmmdc2(b, a%b);  // cmmdc(a,b) = cmmdc(b, restul a/b)
}

7. Recursivitate pe tablouri

// Suma elementelor unui vector recursiv
int suma_v(int v[], int n) {
    if (n == 0) return v[0]; // cazul de baza: un singur element
    else return v[n] + suma_v(v, n-1);
}

// Afisare inversa a vectorului
void afisare_inversa(int v[], int n) {
    if (n == 0) { cout << v[0] << " "; return; }
    cout << v[n] << " ";      // afisam ultimul
    afisare_inversa(v, n-1);  // continuam cu restul
}

int main() {
    int a[] = {4, 5, 7, 8};
    cout << suma_v(a, 3);     // 4+5+7+8 = 24
    afisare_inversa(a, 3);   // 8 7 5 4
}
GREȘELI FRECVENTE LA RECURSIVITATE
  • ❌ Uiți cazul de bază → bucla infinită, program crash (stack overflow)
  • ❌ Cazul recursiv nu "scade" spre cazul de bază
  • ❌ Parametrul nu se modifică în apelul recursiv (ex: fib(n) în loc de fib(n-1))
RECURSIV vs ITERATIV

Orice funcție recursivă poate fi rescrisă iterativ (cu bucle) și invers. Recursivitatea e mai elegantă dar folosește mai multă memorie (stivă). Pentru teste, amândouă sunt corecte.

Probleme din culegere — rezolvate pas cu pas

Exercițiul 1 § 1 Operații pe un număr întreg cu funcții

Enunț: De la tastatură se citește un număr întreg n (n<100000). Elaborați un program cu funcții care determină: a) Numărul de cifre; b) Suma cifrelor; c) Cifra maximă; d) Cifra minimă; e) Inversul numărului.

Planificăm funcțiile: Vom crea câte o funcție pentru fiecare operație.
Cheia pentru cifre: Extragem cifrele cu n%10 (ultima cifră) și n/10 (restul numărului).
Scriem codul: 
#include <iostream>
using namespace std;

// a) Numarul de cifre
int nrCifre(int n) {
    if (n < 0) n = -n;   // tratam si numerele negative
    if (n == 0) return 1;  // 0 are o cifra
    int cnt = 0;
    while (n > 0) {
        cnt++;       // numaram fiecare cifra
        n = n / 10; // eliminam ultima cifra
    }
    return cnt;
}

// b) Suma cifrelor
int sumaCifre(int n) {
    if (n < 0) n = -n;
    int s = 0;
    while (n > 0) {
        s += n % 10;  // adunam ultima cifra
        n = n / 10;   // eliminam ultima cifra
    }
    return s;
}

// c) Cifra maxima
int cifrMax(int n) {
    if (n < 0) n = -n;
    int maxC = 0;  // cifra maxima gasita
    while (n > 0) {
        int c = n % 10;    // ultima cifra
        if (c > maxC) maxC = c; // actualizam daca e mai mare
        n = n / 10;
    }
    return maxC;
}

// d) Cifra minima
int cifrMin(int n) {
    if (n < 0) n = -n;
    int minC = 9;   // incepem cu maximul posibil
    while (n > 0) {
        int c = n % 10;
        if (c < minC) minC = c;
        n = n / 10;
    }
    return minC;
}

// e) Inversul numărului
int invers(int n) {
    if (n < 0) n = -n;
    int inv = 0;
    while (n > 0) {
        inv = inv * 10 + n % 10; // construim inversul cifra cu cifra
        n = n / 10;
    }
    return inv;
}

int main() {
    int n;
    cout << "n= "; cin >> n;
    cout << "Nr. cifre: "    << nrCifre(n)    << endl;
    cout << "Suma cifrelor: " << sumaCifre(n)  << endl;
    cout << "Cifra maxima: "  << cifrMax(n)    << endl;
    cout << "Cifra minima: "  << cifrMin(n)    << endl;
    cout << "Invers: "        << invers(n)     << endl;
    return 0;
}

// CMMDC (cel mai mare divizor comun)
int cmmdc(int a, int b) {
    while (b != 0) {
        int r = a % b; // calculam restul
        a = b;          // a devine b
        b = r;          // b devine restul
    }
    return a; // cand b=0, a este cmmdc
}

// CMMMC (cel mai mic multiplu comun)
long cmmmc(int a, int b) {
    return (long)a / cmmdc(a, b) * b; // formula: a*b/cmmdc(a,b)
}

Exercițiul 5 § 4 (recursiv) Serii matematice recursive

Enunț: Calculați recursiv: a) 1+3+5+...+(2n-1), b) 2+4+6+...+2n, g) -1/2 + 1/4 - 1/8 + 1/16 - ... + (-1)^n × 1/2^n

#include <iostream>
#include <cmath>  // pentru pow()
using namespace std;

// a) Suma numerelor impare: 1+3+5+...+(2n-1)
int suma_impare(int n) {
    if (n == 1) return 1;          // caz de baza: 2*1-1 = 1
    return (2*n-1) + suma_impare(n-1); // termenul curent + suma prec.
}

// b) Suma numerelor pare: 2+4+6+...+2n
int suma_pare(int n) {
    if (n == 1) return 2;       // caz de baza: 2*1 = 2
    return 2*n + suma_pare(n-1); // 2*n + suma pana la n-1
}

// g) -1/2 + 1/4 - ... + (-1)^n * 1/2^n
double serie_alterna(int n) {
    if (n == 1) return -0.5;      // caz de baza: -1/2
    return pow(-1, n) / pow(2, n) + serie_alterna(n-1);
}

int main() {
    int n;

    cout << "a) Suma impare: " << suma_impare(n) << endl;
    cout << "b) Suma pare: "   << suma_pare(n)   << endl;
    cout << "g) Serie: "       << serie_alterna(n) << endl;

}
Referință rapidă

Cheat Sheet — Tot ce trebuie să știi

Funcție cu tip

int nume(int a, int b) { return a+b; }

Returnează o valoare. Apel: x = nume(5, 3);

Funcție void (procedură)

void nume(int a) { cout << a; }

Nu returnează. Apel: nume(5); — linie separată

Parametru prin VALOARE

void f(int x) { x = 100; }

Variabila originală NU se schimbă. x este copie.

Parametru prin REFERINȚĂ

void f(int &x) { x = 100; }

Variabila originală SE SCHIMBĂ. & = adresă.

Vector ca parametru

void f(int v[], int n) { v[0]=1; }

Array-urile se transmit MEREU prin referință!

Matrice ca parametru

const int M=100; void f(int t[M][M], int n, int m)

Trebuie specificat nr. de coloane (sau constanta).

Șir de caractere

void f(char s[]) { int n=strlen(s); }

Include <cstring> pentru strlen().

Prototip (declarație anticipată)

int suma(int, int);

Scris înainte de main() dacă funcția e definită după.

Variabilă globală

int g = 0; // in afara oricărei funcții

Vizibilă în tot programul. Init. cu 0 automat.

Recursivitate — model

int f(int n) { if(n==0) return 1; return n*f(n-1); }

Când folosesc void vs cu tip?

Funcție CU TIP când...

  • Calculezi și ai nevoie de un rezultat: maxim, suma, factorial
  • Vrei să folosești rezultatul într-o expresie
  • Vrei să combini apeluri: suma(a, maxim(b,c))

Funcție VOID când...

  • Afișezi ceva pe ecran
  • Citești date cu cin
  • Modifici mai mulți parametri (prin referință)
  • Sortezi, inversezi un vector

A. Funcții — tot ce trebuie să știi

Cum funcționează apelul unui subprogram? (pe stivă)

EXPLICAȚIE VIZUALĂ

Când apelezi o funcție, programul:

  1. Oprește execuția în main()
  2. Salvează adresa de revenire pe stivă
  3. Copiază parametrii actuali în parametrii formali (pe stivă)
  4. Execută corpul funcției
  5. Returnează rezultatul și revine la locul apelului

Parcurgerea matricei — modelul de bază

REGULA DE AUR

La o matrice n×m: linia i merge de la 0 la n-1, coloana j de la 0 la m-1. Elementul de pe linia i, coloana j este t[i][j].

const int MAX = 100;

// Suma elementelor pe fiecare linie
void sume_linii(int t[MAX][MAX], int n, int m) {
    for (int i=0; i<n; i++) {
        int s = 0;
        for (int j=0; j<m; j++) s += t[i][j];
        cout << "Suma liniei " << i << ": " << s << endl;
    }
}

// Suma elementelor de pe perimetrul matricei
int suma_perimetru(int t[MAX][MAX], int n, int m) {
    int s = 0;
    for (int j=0; j<m; j++) s += t[0][j] + t[n-1][j]; // prima si ultima linie
    for (int i=1; i<n-1; i++) s += t[i][0] + t[i][m-1]; // prima si ultima col
    return s;
}

// Interschimbare doua linii
void schimba_linii(int t[MAX][MAX], int m, int a, int b) {
    for (int j=0; j<m; j++) {
        int temp = t[a][j];
        t[a][j] = t[b][j];
        t[b][j] = temp;
    }
}

bool estePrim(int n) {
    if (n < 2) return false; // 0 si 1 nu sunt prime
    for (int i = 2; i * i <= n; i++) // mergem pana la radacina patrata din n
        if (n % i == 0) return false; // gasit divizor → nu e prim
    return true; // niciun divizor gasit → este prim
}

int suma_cifre_rec(int n) {

int cifra_max_rec(int n) {
    if (n < 10) return n;  // o singura cifra — ea insasi e maxima
    int ultima = n % 10;
    int max_rest = cifra_max_rec(n / 10);
    return (ultima > max_rest) ? ultima : max_rest;
}
